Cover ALL of these scenarios (from CONTEXT.md requirements):
1. **Normal cooldown flow:** Cast spell with >1.5s CD -> icon shows -> CD expires -> icon hides
2. **GCD-only state:** Cast instant spell -> GCD fires -> icon should NOT hide during GCD
3. **Charge spell (charges remaining):** Use one charge of multi-charge spell -> icon shows (recharging)
4. **Charge spell (all charges full):** All charges available -> icon hides
5. **Charge spell (last charge used):** Use final charge -> icon shows
6. **Combat lockdown:** Player enters combat -> cooldown events still fire -> verify visibility updates work in combat (SetAlpha is not restricted)
7. **Loading screen (PLAYER_ENTERING_WORLD):** Zone transition -> event fires -> visibility re-synced
8. **Spec change (PLAYER_SPECIALIZATION_CHANGED):** Change spec -> spell IDs may change -> icons with old spells get nil from GetSpellInfo -> fail-show (visible)
9. **Talent swap mid-session:** Swap talents -> spells change -> next SPELL_UPDATE_COOLDOWN re-evaluates
10. **Feature toggle on:** User enables checkbox -> RefreshHideWhenOffCooldown -> UpdateAllViewers -> icons update
11. **Feature toggle off:** User disables checkbox -> RefreshHideWhenOffCooldown -> all icons restored to alpha=1
12. **Feature disabled, then layout refresh:** RefreshLayout hook fires but featureEnabled=false -> icons stay alpha=1
13. **Invalid/nil spellID on icon:** icon.cooldownInfo exists but spellID is nil -> fail-show (visible)
14. **Icon with no cooldownInfo:** New/empty icon frame -> skipped entirely (no crash)
15. **Addon load order:** Init -> LoadAddOn -> SkinCooldownManager -> EnableHideWhenOffCooldown -> hooks set
16. **Multiple bar types:** Essential enabled, Utility disabled -> only Essential icons hide
17. **Short cooldown spell (0.75-1.5s):** If isOnGCD is nil, our fallback treats this as GCD -> fail-show (acceptable? document trade-off)
18. **Secret Value on duration (Midnight):** cdInfo.duration is SecretValue -> arithmetic comparison throws Lua error
